# Chrome DevTools 150: AIエージェント連携とCSS編集機能の強化

## 概要

Chrome 150に合わせて提供されたDevToolsのアップデートでは、CSSの`@container`ルールや`@function`ルールをDevTools上で直接編集できるようになったほか、Performance・Network・Sources各パネルのウィジェットにGemini(AIエージェント)が参照するデータが追加され、AIによるデバッグ支援がさらに拡張された。あわせてStylesタブなどのスクリーンリーダー対応も改善されている。

## 背景

Chromeはここ数年、DevTools内にAIアシスタント(Gemini)を組み込み、パフォーマンス解析やネットワークリクエストの調査を自然言語で行えるようにする取り組みを進めてきた。Chrome 150では、AIエージェントがツール呼び出しを通じて参照できるデータの範囲がさらに広がり、Lighthouse・Network・Sources・Performanceの4パネルにまたがる9個のウィジェットが対象に加わった。同時に、レスポンシブデザインで多用される`@container`クエリや、CSSの計算ロジックを再利用可能にする`@function`ルールについても、DevTools上でのインプレース編集がサポートされた。

## なぜ重要なのか

- **AI支援デバッグの実用性向上**: AIエージェントが参照できるデータソースが増えることで、「このページが遅い理由を教えて」といった曖昧な質問に対しても、より正確な根拠に基づいた回答が得られやすくなる。
- **レスポンシブ開発の効率化**: `@container`ルールをDevTools上で直接編集・実験できるようになったことで、コンテナクエリを使ったレイアウト調整のイテレーションが速くなる。
- **CSSの抽象化がGUIでも扱いやすくなる**: `@function`はCSSに計算ロジックの再利用性をもたらす比較的新しい仕様だが、DevToolsでの編集サポートにより実務での採用ハードルが下がる。
- **アクセシビリティの継続的な改善**: Stylesタブのタブ選択やセクション移動時のスクリーンリーダー読み上げが改善され、DevTools自体の利用者体験も向上している。

## 実務での活用例

- Performanceパネルでボトルネックを調査する際、Geminiに「このタスクが重い原因」を尋ねると、拡張されたウィジェットデータをもとにネットワークやソースコードの情報まで踏まえた回答を得られる。
- レスポンシブレイアウトのブレークポイント調整時、Elementsパネルから`@container`のクエリ条件をその場で書き換えて即座に見た目を確認できるため、CSSファイルとブラウザを往復する手間が減る。
- 共通の計算式(例: 余白やフォントサイズのスケーリング)を`@function`として定義しているプロジェクトで、DevTools上から関数の中身を直接調整しながらデザイン確認ができる。

## 注意点

- DevToolsのAI機能(Gemini)はデータ収集・送信を伴うため、機密情報を含むページやリクエストを扱う際は、社内のセキュリティポリシーやプライバシー要件を確認してから利用すること。
- `@container`や`@function`はモダンブラウザでの対応が前提となる比較的新しいCSS機能であり、対象ユーザーのブラウザバージョンによっては期待通りに動作しない可能性があるため、フォールバックの検討が必要。
- DevTools上での編集はあくまで一時的な確認用であり、実際の変更はソースコード側にも反映させる必要がある。
