# CSS `polygon()`の角丸指定と`AccentColor`システム色 ― Edge/Chrome 150で追加されたCSS新機能

## 概要

2026年7月にリリースされたMicrosoft Edge 150(および同時期のChrome 150)の web platform アップデートで、CSSの`polygon()`シェイプ関数に角を丸められる`round`パラメータが追加されたほか、OS側で設定されたアクセントカラーをCSSの色値として直接参照できる`AccentColor`・`AccentColorText`のシステムカラーが利用可能になった。どちらもベンダー非依存のCSS標準機能であり、これまでJavaScriptや複雑な計算に頼っていた表現を宣言的なCSSだけで実現できるようになる。

## 背景

`clip-path: polygon()`は任意の多角形を描画できる便利な機能だったが、角を丸めるには各頂点にベジェ曲線を手計算で挿入する必要があり、実務上の障壁になっていた。Edge/Chrome 150では`polygon(round 50px, 50% 0%, 0% 100%, 100% 100%)`のように`round`キーワードと丸め半径を指定するだけで、角丸の多角形を宣言的に記述できるようになった。

また、OSのテーマ設定で選べる「アクセントカラー」は、これまでネイティブUIコンポーネント(フォーム部品など)にしか反映されず、Web開発者がCSSから直接参照する手段がなかった。今回追加された`AccentColor`(背景色相当)と`AccentColorText`(前景色相当)というシステムカラーキーワードにより、`color`や`background-color`プロパティで`AccentColor`を指定すれば、ユーザーのOS設定に連動した配色をWebページ側でも再現できる。

## なぜ重要なのか

- **JavaScript不要の宣言的な角丸表現**: これまでJSやSVGフィルタで代替していた角丸多角形の表現が、CSSのみで完結するようになり、実装・保守コストが下がる。
- **OSとの一貫したビジュアル統合**: `AccentColor`を使うことで、ユーザーが普段使っているOSのテーマカラーとWebアプリの配色を自然に連動させられ、ネイティブアプリのような統一感のあるUIを作りやすくなる。
- **ベンダー間の足並みが揃いつつある標準機能**: ChromeとEdge(同じChromiumベース)がほぼ同時にサポートしたことで、今後他のブラウザへの波及も期待でき、実務投入の判断がしやすくなる。
- **アクセシビリティ・パーソナライゼーションの後押し**: ユーザー自身が選んだ配色をサイト側が尊重する設計は、視認性の好みに配慮したUI作りにもつながる。

## 実務での活用例

- バッジやタグ、装飾的なアイコン背景など、角丸多角形をデザインカンプ通りに実装したい場面で、`clip-path: polygon(round 8px, ...)`を使えばSVGアセットを用意せずCSSだけで再現できる。
- 「選択中」を示すハイライト色やフォーカスリングの色に`background-color: AccentColor`を指定し、ユーザーのOSテーマ(例: Windowsのアクセントカラー設定)に自動で追随するUIを実装する。
- ダークモード/ライトモード対応と組み合わせて、`AccentColorText`をボタンの文字色に使うことで、背景の`AccentColor`に対して適切なコントラストを保った配色をブラウザ側の実装に委ねられる。

## 注意点

- 執筆時点ではChromium系ブラウザ(Chrome/Edge)での対応が中心であり、Safari・Firefoxでの対応状況は個別に確認が必要。非対応ブラウザ向けにはフォールバックの配色・角丸実装(従来通りのベジェ曲線指定やボーダー半径)を用意しておくこと。
- `AccentColor`はユーザーのOS設定に依存するため、ブランドカラーを厳密に統一したい要素に安易に使うと、環境によって配色が意図せず変化する点に注意が必要。
- `polygon()`の`round`はあくまで直線区間の角に対する丸めであり、複雑な曲線形状すべてを代替できるわけではない点を理解した上で使い分けること。
