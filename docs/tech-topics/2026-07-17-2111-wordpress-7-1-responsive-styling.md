# WordPress 7.1がベータ入り ― CSS不要のレスポンシブスタイリングとコラボレーション機能を強化

## 概要

WordPress 7.1のBeta 1が2026年7月15日に公開され、以降は週次でベータ版がリリースされる予定。正式リリースは2026年8月19日で、米国で開催されるWordCamp USに合わせたタイミングとなっている。今回の目玉は、カスタムCSSを書かずにブロックのレスポンシブ表示やホバー・フォーカスなどのインタラクティブ状態を制御できるスタイリング機能の追加で、あわせてPlaylistブロックやTabsブロックなどの新規ブロック、@メンション対応のNotes機能によるコラボレーション強化、HEIC/AVIF/UltraHDRなどメディア処理まわりの改善も予定されている。

## 背景

WordPressはブロックエディタ(Gutenberg)を中心に、テーマやプラグインを問わず「コードを書かずにサイトを作れる」体験の拡張を継続的に進めてきた。これまでレスポンシブ対応やホバー時の見た目変更といった表現は、テーマのカスタムCSSやプラグインに頼らざるを得ない場面が多かった。7.1では、この領域をGlobal Stylesおよび個別ブロックの設定画面から標準機能として扱えるようにし、`theme.json`でテーマ作成者が独自のレスポンシブブレークポイントを定義できるようにするなど、テーマ側の柔軟性も広げる方向で開発が進められている。

## なぜ重要なのか

レスポンシブスタイリングやインタラクティブ状態のスタイリングが標準UIから設定できるようになると、これまでカスタムCSSやテーマ改修が必要だった調整を、エディタ操作だけで完結できるようになる。これはノーコード・ローコードでサイト運用をしている制作者やクライアントワークを行う開発者にとって、保守コストの削減に直結する。また、REST APIの添付ファイル(attachment)エンドポイントに出力フォーマットやプログレッシブレンダリング、品質パラメータが追加されるなど、ヘッドレスWordPressやカスタムフロントエンドを構築する開発者にとっても影響がある変更が含まれている。

## 実務での活用例

- クライアント案件でボタンのホバー・フォーカス時の配色変更を求められた際、カスタムCSSを追加せずGlobal Styles/ブロック設定のインタラクティブ状態コントロールで対応する。
- テーマ開発時に`theme.json`で独自のレスポンシブブレークポイントを定義し、サイト固有のデザインシステムに合わせた表示切り替えを実装する。
- 音声コンテンツを扱うメディアサイトで、新設されるPlaylistブロック(波形ビジュアライザ付き)を使い、カスタムプラグインなしで再生リストUIを構築する。
- 複数人でのコンテンツレビューが多い編集チームで、@メンション対応になったNotes機能を使い、コメントのやり取りを効率化する。
- ヘッドレスWordPress構成でメディアを配信している場合、REST attachmentエンドポイントに追加される出力フォーマット・品質パラメータを利用し、フロントエンド側での画像最適化処理を簡略化する。

## 注意点

- 記事執筆時点(Beta 1公開直後)の情報であり、正式リリース(2026年8月19日予定)までに仕様や搭載機能が変更される可能性がある。本番環境への適用前に必ずリリースノートを確認すること。
- 新しいスタイリング機能や`theme.json`のブレークポイント定義は、既存テーマとの互換性を崩す可能性があるため、ステージング環境での検証を挟むことが望ましい。
- HEIC/AVIF/UltraHDRなど新しいメディアフォーマット対応はクライアントサイド処理に依存する部分があり、対象ブラウザやサーバー環境によっては挙動が異なる場合がある。
