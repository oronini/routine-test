# Claude Code の「Routines」機能 ― スケジュール実行されるクラウドAIエージェント

## 概要

Anthropic は Claude Code に「Routines（ルーティン）」という新機能をリサーチプレビューとして追加した。Routines は、あらかじめ定義したプロンプト・接続先リポジトリ・利用可能なツールをひとまとめにした「保存済みタスク」を、スケジュール（毎時・毎日・平日・毎週など）、API呼び出し、あるいは GitHub の Webhook イベントをトリガーとして、Anthropic のクラウド環境上で自動的に実行できる仕組みである。ユーザーがアクティブなセッションを開いていなくても、バックグラウンドでエージェントが作業を継続できる点が特徴である。

## 背景

これまでの Claude Code は、開発者がターミナルやIDE上で対話的に操作することを前提としたツールだった。しかし実際の開発現場では、CIの監視、PRレビューへの追従、定期的なリポジトリの棚卸しや依存関係チェックなど、「人が張り付いていなくても定期的に自動で回したい」作業が数多く存在する。Routines はこうしたニーズに応える形で、Claude Code をターミナル上のアシスタントから、クラウド上で自律的に稼働する運用エージェント基盤へと拡張するものである。Pro・Max・Team・Enterprise の各有料プランで利用可能で、web版（claude.ai/code）やCLIの `/schedule` コマンドから設定できる。

## なぜ重要なのか

Routines の登場は、AIコーディングエージェントの位置づけが「開発者を補助するチャットツール」から「継続的に稼働するインフラの一部」へと変わりつつあることを示している。GitHub Actions のような既存のCI/CD自動化基盤と組み合わせることで、コードのプッシュやIssueの作成といったイベントに応じてAIエージェントが自律的に反応し、判断を伴う作業（例: リリースノートの下書き作成、ドキュメントの定期更新、失敗したテストの一次切り分け）まで担えるようになる。単純なスクリプトによる自動化と異なり、状況に応じた文脈理解や判断が求められるタスクを自動化できる点が、この機能の本質的な価値である。

## 実務での活用例

- 毎朝リポジトリの依存関係やセキュリティアドバイザリを確認し、問題があればIssueを自動起票する
- GitHub Webhookをトリガーに、PRが作成されるたびにコーディング規約やドキュメント更新の抜け漏れをチェックする
- APIエンドポイント経由で外部の監視システムと連携し、障害検知時に一次調査レポートを自動生成する
- 定期的にドキュメントディレクトリを巡回し、内容の重複や古い記述がないかをレビューする（本記事もそうした自動生成の一例である）

## 注意点

- 現時点ではリサーチプレビュー段階であり、プランごとに1日あたりの実行回数上限が設けられている（Pro: 5回/日、Max: 15回/日、Team/Enterprise: 25回/日など）
- クラウド環境で自律的にコードやリポジトリを操作するため、権限スコープの設計や、意図しない変更・破壊的操作を防ぐガードレールの整備が重要になる
- Webhookやスケジュールで起動する性質上、想定外の頻度・タイミングでの実行や、外部入力（Issue本文やコメントなど）を介したプロンプトインジェクションのリスクにも留意する必要がある
