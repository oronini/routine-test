# TypeScript 7.0：Go言語ネイティブコンパイラへの移行

## 概要

2026年6月18日、MicrosoftはTypeScript 7.0のリリース候補（RC）を公開した。最大の特徴は、これまでTypeScript自身（JavaScript）で実装されていたコンパイラを、Go言語に移植した点にある。開発コードネーム「Project Corsa」として2025年3月に発表されて以来進められてきたこの取り組みにより、型チェックの速度が既存実装比でおよそ10倍に向上したと報告されている。従来のJavaScript実装は「Strada」と呼ばれ、当面は両実装が並行して扱われる見込みである。

## 背景

TypeScriptの利用が拡大するにつれ、大規模なコードベースにおけるコンパイル・型チェックの速度がボトルネックとして顕在化していた。JavaScript（Node.js）上で動作する従来のコンパイラは、シングルスレッド実行や動的言語特有のオーバーヘッドにより、数百万行規模のプロジェクトでは型チェックに1分以上を要することも珍しくなかった。これに対しMicrosoftは、既存の型チェックの仕様やセマンティクスを維持したまま、実装言語だけをGoに置き換える「移植（ポート）」というアプローチを選択した。単なる書き直しではなく既存の型システムの互換性を保つことを重視した結果、VS Codeのコードベース（約150万行）での型チェックが約78秒から約7.5秒へと大幅に短縮されたと報告されている。

## なぜ重要なのか

コンパイル・型チェックの速度は、エディタ上での補完やエラー表示の応答性、CIパイプラインの実行時間、ひいては開発者の生産性に直結する。特に大規模なモノレポやエンタープライズ規模のプロジェクトでは、型チェックの待ち時間が開発体験そのものを左右してきた。TypeScript 7.0のネイティブコンパイラは、既存の型チェック結果との互換性を保ちながら性能を大幅に改善するため、多くのプロジェクトが比較的低リスクで恩恵を受けられる点が重要である。また、`--incremental`やプロジェクト参照、`--build`モードといった主要機能も移植が進んでおり、既存のビルド構成を大きく変えずに移行できる可能性が高いことも注目される理由の一つである。

## 実務での活用例

- 大規模なTypeScriptプロジェクトやモノレポにおいて、CI上のビルド・型チェック時間を計測し、TypeScript 7.0 RC（`tsgo`）へのアップグレードによる短縮効果を事前に検証する。
- VS CodeなどのエディタでTypeScript 7系のプレビュー機能を有効化し、日常の開発における補完・エラー表示のレスポンス改善を体感的に確認する。
- 既存のビルド設定（`tsconfig.json`のプロジェクト参照構成など）が新しいネイティブコンパイラでも問題なく動作するか、ステージング環境で先行検証する。
- チーム内でGA（正式版）リリースのタイムラインを共有し、段階的な移行計画（一部パッケージからの先行導入など）を検討する。

## 注意点

- 現時点（2026年7月）ではRC（リリース候補）段階であり、正式なGAリリースはRCから約1か月後を見込むとされているものの、確定した日付ではない点に注意が必要である。
- 移植であるため型チェックのセマンティクスは維持される方針だが、実装言語の変更に伴うエッジケースでの挙動差異が生じる可能性があり、本番導入前には十分な回帰テストが推奨される。
- サードパーティのビルドツールやエディタ拡張機能が新しいネイティブコンパイラに対応しているかどうかも、移行前に確認しておく必要がある。
