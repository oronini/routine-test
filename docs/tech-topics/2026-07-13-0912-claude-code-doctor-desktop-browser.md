# Claude Codeの週次アップデート：デスクトップ内蔵ブラウザと自己修復型「/doctor」

## 概要

Anthropicは2026年7月6日〜10日（Week 28、バージョンv2.1.202〜v2.1.206）のアップデートで、Claude Code Desktopアプリに内蔵ブラウザ機能を追加するとともに、セットアップ診断コマンド`/doctor`を単なるレポート出力から「問題を検出し、確認を取った上で自動修正する」ツールへと刷新した。あわせてAuto Modeの安全性強化や、`claude agents`のエージェントビュー表示の改善など、開発体験全般に関わる複数の改善が行われている。

## 背景

Claude Codeはターミナル上で動作するAIコーディングエージェントとして急速に機能拡張を続けており、週次の「What's new」ダイジェストという形で頻繁にアップデートが公開されている。今回のWeek 28では、これまでローカルの開発サーバーのプレビューに限られていたブラウザ的な操作能力を、外部サイト全般に広げた点が大きな変化である。あわせて、AIエージェントが自律的にコードベースやツール構成を管理する上で欠かせない「セルフメンテナンス」機能として、`/doctor`（エイリアス`/checkup`）が強化された。

## なぜ重要なのか

- **内蔵ブラウザ**: Claude Code Desktopがサンドボックス化されたブラウザを持つことで、ドキュメントやデザインカンプなど外部サイトの内容を読み取り、クリック操作を含めて対話できるようになった。ローカルの開発サーバーをプレビューする感覚で、外部情報の参照や検証をエージェントに任せられる。ブラウジングセッションの永続化可否を選べ、外部サイト上のアクションは安全性分類器によってレビューされるため、無制限な自律操作にはなっていない。
- **`/doctor`の自己修復化**: 従来は読み取り専用のレポートを出すだけだったコマンドが、インストールの健全性チェック、使われていないスキル・MCPサーバー・プラグインのコンテキストコスト評価、チェックイン済みの`CLAUDE.md`との重複排除、遅いフックの検出などを行い、確認を取った上で実際に修正まで行うようになった。エージェントの運用コストやコンテキスト消費を継続的に最適化する仕組みが、コマンド一つで手に入る。
- **Auto Modeの安全性強化**: セッションのトランスクリプトファイルへの改ざんをブロックし、コンテキストから解決できない変数に対する`rm -rf`実行前には確認を求めるようになった。自律実行の範囲が広がるほど、こうした事故防止のガードレールが重要になる。
- **バックグラウンドタスク通知の信頼性向上**: バックグラウンドタスクの通知が「人間からの入力は発生していない」ことを明示するようになり、会話ログ内に偽装された承認文言があっても、それを実際の承認として扱わないようにする防御が入った。プロンプトインジェクション対策の一環として実務上の意味が大きい。

## 実務での活用例

- ローカルにモックしていない外部APIドキュメントやFigmaのデザインカンプを、Claude Code Desktopの内蔵ブラウザで直接参照させながら実装を進める。
- 大規模なリポジトリで運用しているエージェント設定（スキル、MCPサーバー、プラグイン、`CLAUDE.md`）が肥大化してきたら、`/doctor`（または`/checkup`）を定期的に実行し、不要な設定やコンテキストコストの高い項目を棚卸しする。
- Auto Modeを有効にした自律実行パイプラインを運用しているチームは、今回のトランスクリプト改ざん防止や`rm -rf`確認強化を踏まえ、既存のガードレール設定やレビュー体制を見直す。
- CIやスケジュール実行されるルーティン（本ドキュメントを生成しているような自動化タスク含む）では、バックグラウンド通知に「人間の入力は発生していない」旨が明記されるようになったことを前提に、ログ内の偽の承認表現を信用しないようエージェント側のプロンプト設計を確認する。

## 注意点

- 内蔵ブラウザは安全性分類器によるレビューを経るとはいえ、外部サイト上での操作を許可する範囲は明示的に設計・確認する必要がある。機密情報を含むページへのアクセスを許可する場合は特に注意する。
- `/doctor`による自動修正は確認プロンプトを挟むとはいえ、`CLAUDE.md`のトリミング提案などはチームの意図的な記述を削ってしまう可能性があるため、提案内容を都度レビューすることが望ましい。
- 本記事の内容は2026年7月13日時点の公開情報に基づく。Claude Codeは週次で機能更新が続いているため、最新の挙動は公式ドキュメントで随時確認すること。
