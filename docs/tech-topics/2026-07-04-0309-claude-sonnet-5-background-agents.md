# Claude Sonnet 5への刷新とバックグラウンドエージェントの自律化

## 概要

2026年6月末から7月初頭にかけて、AnthropicはClaude Codeにおけるデフォルトモデルを新しい「Claude Sonnet 5」へと切り替えた。Claude Sonnet 5は100万トークンのコンテキストウィンドウをネイティブにサポートしており、2026年8月31日まではプロモーション価格（入力$2／出力$10 per 1Mトークン）で利用できる。これに合わせてClaude Code本体もv2.1.197〜v2.1.199へと段階的にアップデートされ、バックグラウンドエージェントが作業完了時に自動的にコミット・プッシュ・ドラフトPRの作成までを行う機能や、ネットワーク断・レート制限まわりの信頼性向上が図られた。

## 背景

Claude Codeはこの数ヶ月、単発のCLIツールから「複数のバックグラウンドエージェントを並行稼働させる開発基盤」へと役割を広げてきた。2026年5月頃から`claude agents`によるエージェントビューやバックグラウンドサブエージェントの許可プロンプト表示改善などが順次導入されており、Claude Sonnet 5の投入とバックグラウンドエージェントの自動化強化は、その延長線上にある取り組みである。特にv2.1.198では、`claude agents`経由で起動したバックグラウンドエージェントがコード作業を終えた際に、人手を介さずコミット・プッシュ・ドラフトPRのオープンまで完了させるようになった点が大きな変化である。

## なぜ重要なのか

これまでのAIコーディングエージェントは、コードの生成・編集までを担い、コミットやPR作成といった「仕上げ」の作業は人間が行うのが一般的だった。バックグラウンドエージェントが自律的にコミット・プッシュ・PRオープンまで完結できるようになったことで、開発者は複数のタスクを並行してエージェントに委任し、結果をPRという形でレビューするだけで済むようになる。またコンテキストウィンドウが100万トークンに拡張されたことで、大規模なモノレポや長い会話履歴を持つセッションでも要約による情報欠落を抑えたまま作業を継続できる点も、実務上のインパクトが大きい。

## 実務での活用例

- 複数の独立したバグ修正や小規模機能追加をバックグラウンドエージェントに並行して依頼し、完成したものから順にドラフトPRとしてレビューキューに積む運用。
- 大規模なコードベースの調査・リファクタリング作業で、100万トークンのコンテキストを活かして関連ファイルを要約せずに一括して読み込ませる。
- CI連携と組み合わせ、エージェントが作成したドラフトPRに対して自動でレビューコメントやCIチェックを回し、人間はマージ判断のみに集中する。

## 注意点

- バックグラウンドエージェントが自動でコミット・プッシュ・PR作成まで行うため、リポジトリ側のブランチ保護やレビュー必須設定を事前に整えておかないと、意図しない変更がpushされるリスクがある。
- プロモーション価格は2026年8月31日までの期間限定であり、恒久的なコスト試算には注意が必要。
- 新モデル・新バージョンへの移行期は、旧バージョンとの挙動差（レート制限やネットワーク再試行の扱いなど）が生じやすいため、重要なワークフローでは段階的な検証が望ましい。
